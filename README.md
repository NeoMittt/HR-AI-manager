# HR Telegram Bot

AI-бот для первичного найма в Telegram с веб-админкой для работодателя. Проект принимает резюме, проводит короткое интервью по вакансии, формирует итоговую сводку по кандидату и помогает работодателю быстро принимать решение.

## Что умеет проект

- приветствует кандидата и не считает первое сообщение резюме;
- показывает выбор из нескольких открытых вакансий;
- принимает резюме текстом или PDF;
- анализирует резюме через OpenRouter;
- отсеивает явно нерелевантных кандидатов по профессии;
- задаёт вопросы по недостающей информации из резюме;
- умеет делать follow-up, если ответ расплывчатый;
- считает score кандидата по критериям:
  - опыт;
  - навыки;
  - мотивация;
  - cultural fit;
- сохраняет итоговую `.txt`-сводку с учётом резюме и интервью;
- показывает полную историю переписки в админке;
- позволяет писать кандидату от имени бота из админ-панели;
- поддерживает статусы сессий, включая `Ждёт решения`;
- отправляет работодателю Telegram-уведомления с кнопками;
- отправляет кандидату уведомление об одобрении или отклонении;
- экспортирует кандидатов в CSV, совместимый с Excel;
- показывает сравнительную таблицу кандидатов по score.

## Основные сценарии

### Для кандидата

1. Кандидат нажимает `/start`.
2. Бот предлагает выбрать вакансию.
3. Бот просит прислать резюме.
4. Резюме анализируется и проверяется на релевантность вакансии.
5. Бот задаёт вопросы только по тем пунктам, которых не хватает в резюме.
6. После интервью формируется итоговая summary и score.
7. Сессия переходит в статус `Ждёт решения`.

### Для работодателя

1. Работодатель открывает веб-панель или Telegram-панель.
2. Видит список кандидатов, статусы, score и summary.
3. Может одобрить, отклонить, написать кандидату, сменить статус или удалить сессию.
4. Может выгрузить таблицу кандидатов в CSV.

- полная история переписки;

- статус `Ждёт решения`;

- удаление и сброс сессий;
- просмотр логов и crash-логов;
- настройка модели, вакансий и поведения бота;
- анимации загрузки, чтобы было видно, что интерфейс работает.

## Telegram-функции работодателя

Работодатель получает уведомления о новых и завершённых кандидатах и может прямо в Telegram:

- одобрить кандидата;
- отклонить кандидата;
- открыть summary;
- перейти к сессии.

## Быстрый старт

### 1. Установите зависимости

```bash
pip install -r requirements.txt
```

### 2. Заполните `settings.json`

Минимально нужны:

```json
{
  "telegram_token": "ВАШ_BOT_TOKEN",
  "openrouter_api_key": "sk-or-...",
  "employer_ids": [123456789]
}
```

Полезные параметры:

- `openrouter_model` — основная модель;
- `openrouter_free_models` — список fallback-моделей;
- `interview_questions_count` — базовое количество тем интервью;
- `block_duration_seconds` — время паузы при off-topic;
- `admin_host` и `admin_port` — адрес веб-панели;
- `active_vacancy_preset` — активный пресет вакансии;
- `open_vacancy_keys` — какие вакансии доступны кандидату на старте.

### 3. Запустите проект

Обычный запуск:

```bash
start_hr_bot.bat
```

Если бот уже запущен и нужно именно обновить процесс:

```bash
restart_hr_bot.bat
```

## Как устроена оценка кандидата

После завершения интервью бот сохраняет:

- балл по опыту;
- балл по навыкам;
- балл по мотивации;
- балл по cultural fit;
- общий score;
- порог вакансии;
- вывод, прошёл кандидат порог или нет.

Эта информация отображается и в summary, и в админ-панели.

## Что сохраняется по кандидату

Для каждой сессии проект хранит:

- выбранную вакансию;
- резюме;
- анализ резюме;
- полную историю диалога;
- ответы на интервью;
- итоговый score;
- решение работодателя;
- итоговую summary.

## Статусы сессий

Основные статусы:

- `waiting_vacancy` — ждёт выбор вакансии;
- `waiting_resume` — ждёт резюме;
- `interviewing` — идёт интервью;
- `blocked` — временная пауза;
- `waiting_decision` — интервью завершено, ждёт решения работодателя;
- `completed` — решение принято;
- `rejected` — отклонён;
- `approved` — одобрен.
